# GyroPrompt 
#DogeCoin #Cloud #Blockchain #Scripting #ScriptLanguage #CSharp #SQL #Wolfram

GyroPrompt was intended to be a Win console-based scripting language centered around the GyroCloud and DogeCoin.

Imagine Python but it's a little bloated, Python is also yelling at you in all caps, and there's a PythonCloud where you use DogeCoin for hosting cloud apps, blockchains attached to namespaces (long term plan was to allow sub-cryptos to be launched with a few commands), namespaces for cloud apps, and file storage. That was the end goal for GyroPrompt. And I had made lots of progress.

The GyroCloud was a simple SQL Server that would manage user accounts, namespaces, and file storage for users.
I had a system set up where GyroPrompt would make .gs become a system recognized file type and would automatically run - much like how a .bat or .vbs automatically executes when opened. This feature was mysteriously patched by Microsoft in a security update
I successfully began scaffolding the infrastructure for DogeCoin integrations with user profiles.
GyroPrompt even had a Wolfram integration for encyclopedic data.
I even had a YouTube page dedicated to this project: https://www.youtube.com/channel/UCbhwN6P65XKcoW82n6QaHMw
... and a website, www.gyorosoft.org which is no longer active (domain expired).

I know not everything was coded to be extremely memory efficient. In fact, some of the coding practices I implemented were very very memory inefficient and bloated.
There was still hundreds of hours of work to do.

The reason I am publicizing this and basically soft-retiring, or maybe temporarily shelving, the project as a whole is because it would have become too time consuming to manage and upkeep - especially had the GyroCloud gone from a proof-of-concept to a real working live server (which I really was close to doing, I was frequently shopping and researching for server equipment and server racks). My first major realization was the anonymity of user profiles, having no ID verification requirement, coupled with file storage and crypto could have led to an absolute disaster in terms of moderation and legal problems. I do not have the resources or time to actively moderate and remove content - I work, go to school, have family, ect ect. All it would have taken is for bad pictures or some illegal content to have been put in someone's file storage, distributed, then BAM - I have feds kicking my door down. That's just the GyroCloud. That's not even the updates and patches for the regular code-side of things that would make the language more functional, powerful, and versatile. Keeping up with the user manual updates was also a major time-consuming headache.

This project could be a good start for someone to look over who's interested in how something like this might work.
